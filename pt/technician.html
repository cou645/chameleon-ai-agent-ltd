AI-Guided System Repair

The AI follows a structured surgical protocol — diagnose before touching, verify at every step

# Chameleon AI diagnosis sequence $ lsusb -v | cabeça -80 Barramento 002 Dispositivo 003: ID 0bda:8153 Realtek USB 10/100/1000 LAN $ dmesg | tail -30 [12.445] r8152 2-1:1.0 eth1: renomeado de eth0 $lsmod | grep r8152 r8152 94208 0 $ modinfo r8152 | versão grep versão: v1.12.13

Sequência estruturada de diagnóstico

Hardware → kernel state → driver status → network → storage. The AI runs each step, reads the JSON result, then decides the next action. It never skips steps or makes assumptions.

🏥

Surgical repair protocol

Primeiro não destrutivo (recarregar módulo, reiniciar serviço). Persista antes do risco (configuração do instantâneo). Verifique em cada etapa. Cada ação que requer reinicialização aciona um interruptor do homem morto que define um @reboot retomar o script automaticamente.

🌐

Remote repair over Tailscale

Diagnose and repair machines anywhere on your Tailscale mesh. The AI issues commands on the remote machine via the terminal bridge, reads results and continues — even across reboots using the checkpoint/resume system.

Porteiro de driver USB

Profile any USB device → analyse its protocol → generate a ready-to-install Linux driver

Caminho A

Protocolos padrão

Nenhuma página de fornecedor encontrada. Gera regras do udev + configuração do modprobe. Nenhuma compilação necessária. Funciona com HID, CDC, armazenamento em massa e outras classes padrão.

udevmodprobe
Caminho B

Protocolo de fornecedor parcial

Páginas do fornecedor detectadas (≤2). Gera um daemon de espaço de usuário hidraw/uhid ou plugin ALSA. Executa no espaço do usuário — não é necessária compilação do kernel.

desenharPlug-in ALSA
Caminho C

Protocolo proprietário

Protocolo totalmente personalizado. Gera um módulo de kernel completo com pacote DKMS e suporte de compilação cruzada ARM para Raspberry Pi 3, 4 e 5.

Back-ends de captura multiplataforma

ETW on Windows, IOKit on macOS, usbmon on Linux. Profile on any OS and generate a Linux driver from the resulting profile.json.

🍓

Compilação cruzada do Raspberry Pi

Destinos Makefile cross-aarch64 (Pi 4/5) e cross-armv7 (Pi 3) são incluídos automaticamente no Caminho C. Construa em x86 e implante em ARM.

📦

Embalagem DKMS

Path C drivers are packaged for DKMS so they survive kernel upgrades without manual recompilation. The AI reviews analysis.json and prompts for any remaining TODO stubs.
