AI-Guided System Repair

The AI follows a structured surgical protocol — diagnose before touching, verify at every step

# Chameleon AI diagnosis sequence $ lsusb -v | голова -80 Шина 002 Пристрій 003: ID 0bda:8153 Realtek USB 10/100/1000 LAN $ dmesg | tail -30 [12.445] r8152 2-1:1.0 eth1: перейменовано з eth0 $ lsmod | grep r8152 r8152 94208 0 $ modinfo r8152 | версія grep версія: v1.12.13
🔍

Структурована послідовність діагностики

Hardware → kernel state → driver status → network → storage. The AI runs each step, reads the JSON result, then decides the next action. It never skips steps or makes assumptions.

🏥

Surgical repair protocol

Спочатку неруйнівний (перезавантажити модуль, перезапустити службу). Зберігати до ризику (конфігурація знімка). Перевіряйте на кожному кроці. Кожна дія, яка потребує перезавантаження, запускає перемикач мертвого людини, який встановлює a @reboot автоматично відновити сценарій.

🌐

Remote repair over Tailscale

Diagnose and repair machines anywhere on your Tailscale mesh. The AI issues commands on the remote machine via the terminal bridge, reads results and continues — even across reboots using the checkpoint/resume system.

Портер драйвера USB

Profile any USB device → analyse its protocol → generate a ready-to-install Linux driver

Шлях А

Стандартні протоколи

Сторінок постачальників не знайдено. Генерує правила udev + конфігурацію modprobe. Компіляція не потрібна. Працює з HID, CDC, масовим накопичувачем та іншими стандартними класами.

udevmodprobe
Шлях Б

Частковий протокол постачальника

Виявлено сторінки постачальників (≤2). Створює демон простору користувача hidraw/uhid або плагін ALSA. Працює в просторі користувача — компіляція ядра не потрібна.

hidrawПлагін ALSA
Шлях C

Власний протокол

Повністю індивідуальний протокол. Створює повний модуль ядра з пакетом DKMS і підтримкою крос-компіляції ARM для Raspberry Pi 3, 4 і 5.

Кроссплатформенний сервер захоплення

ETW on Windows, IOKit on macOS, usbmon on Linux. Profile on any OS and generate a Linux driver from the resulting profile.json.

🍓

Перехресна компіляція Raspberry Pi

Цілі Makefile cross-aarch64 (Пі 4/5) і cross-armv7 (Pi 3) автоматично включаються в Path C. Створіть на основі x86 і розгорніть на ARM.

📦

Упаковка DKMS

Path C drivers are packaged for DKMS so they survive kernel upgrades without manual recompilation. The AI reviews analysis.json and prompts for any remaining TODO stubs.
