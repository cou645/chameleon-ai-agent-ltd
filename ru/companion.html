Групповое P2P-видео и голос

Полноценное видео и аудио WebRTC для групп из 4 и более человек — без центрального медиасервера и без ежемесячных счетов. Кран Групповое видео чтобы позвонить всем онлайн-партнерам одновременно. Сетка автоматически адаптируется по мере присоединения участников: 1 = полноэкранный режим, 2 = параллельно, 3 = 2+1, 4 = 2×2, 5+ = прокручиваемая сетка с автоподбором. Ваш местный канал отображается в формате «картинка в картинке». Битрейт уменьшается для каждого узла, чтобы сохранить баланс качества при любом размере группы.

📞

SIM-мост вызовов

Вы уже разговариваете по сотовому телефону и хотите подключить одноранговый узел P2P? Кран Добавить в группу звонков — приложение регистрирует вызов WebRTC в стеке телекоммуникаций Android вместе с вызовом SIM. Android представляет встроенную опцию «Объединить вызовы»; принимая его, микширование звука передается на телефонное оборудование. Объединенный вызов воспроизводится в только наушник — полная конфиденциальность, без громкой связи, никто в комнате не слышит. Если аппаратная конференция не поддерживается оператором связи, мост гарнитуры Bluetooth вместо этого направляет весь звук через канал BT SCO.

Как это соединяется

🔐

Сетка, закрепленная на хвостовой чешуе

Все каналы работают через вашу частную сеть Tailscale. Никакие порты не открыты для общедоступного Интернета. Телефон должен быть авторизован в вашей хвостовой сети — все остальное зашифровано с помощью WireGuard.

🔌

Одиночный мост WebSocket

Рабочий стол работает chameleon_mobile_bridge.py как поток демона внутри процесса Qt. Телефонное приложение подключается к порту 8770 на IP-адресе Tailscale. Одно постоянное соединение передает все каналы в виде типизированных сообщений JSON.

🤖

Термукс на Android

Сигнальный сервер (chameleon_signal.py) может работать непосредственно на телефоне Android через Termux, что делает телефон одновременно клиентом и сигнальным узлом. Для одноранговых соединений WebRTC внутри вашей хвостовой сети не требуется облачный ретранслятор.

📶

Резервный вариант локальной сети

Когда Tailscale недоступен, lan_video.py передает потоковое видео через UDP (порт 47732) и аудио через UDP (порт 47733) непосредственно в локальной сети — те же каналы, без накладных расходов VPN.
