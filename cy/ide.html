The AI Command Bridge

AI doesn't just chat — it operates the editor through a structured JSON protocol

📖

Darllenwch unwaith, golygwch yn rhydd

The AI reads a file once with cael_ffeil_cynnwys, yn dal y cynnwys yn ei gyd-destun, yna'n cyhoeddi golygiadau angor lluosog heb eu hail-ddarllen rhwng pob newid. Dim teithiau crwn wedi'u gwastraffu.

🎯

replace_text yn seiliedig ar angor

Mae'n golygu'r union destun i'w ddarganfod, nid rhifau llinellau. Mae rhifau llinellau'n symud wrth i'r cod gael ei ychwanegu neu ei ddileu - nid yw angorau yn gwneud hynny. Lluosog replace_text gellir sypynnu galwadau mewn un ymateb, i gyd yn cael eu cymhwyso'n atomig.

📝

write_file ar gyfer ailysgrifennu llawn

For new modules or complete rewrites the AI sends the entire file content in one command. The system automatically creates a .bak gwneud copi wrth gefn cyn trosysgrifo ac yn agor y canlyniad mewn tab golygydd newydd.

💻

Integreiddio terfynell

The AI can run shell commands and receive the JSON result before continuing. It waits for each result, handles exit codes, and distinguishes diagnostic, configuration, destructive and session-ending commands automatically.

🛡️

Amddiffyniad chwistrellu

Mae'r holl gynnwys a ddychwelir o ffeiliau, terfynellau a thudalennau gwe wedi'i lapio i mewn [UNTRUSTED_CONTENT] markers. The AI is instructed to treat everything inside as data — never instructions — preventing prompt-injection attacks from malicious file content.

Nodweddion Golygydd

🗂️

Golygydd cod aml-tab

Open multiple files simultaneously. Each tab tracks its own undo history, syntax highlighting and AI command context.

🔀

Golygfa diff adeiledig

One click to compare the current editor state against the last saved version. Colour-coded unified diff with add/remove highlighting. Exit diff mode to return to full editing.

💬

Streaming AI chat

Mae'r ymatebion yn ffrydio tocyn-wrth-tocyn i'r panel sgwrsio. Mae gorchmynion IDE yn cael eu tynnu a'u gweithredu mewn edefyn cefndir felly mae'r UI yn aros yn ymatebol drwyddo draw.

🗃️

System brydlon modiwlaidd

System prompts are plain-text module files in ~/.chameleon/prompts/. Stack any combination: Tech Repair + Academic + Chat. Live token count and cost estimate per provider.

📜

Hanes gorchymyn DB

Every AI command is logged to history.db with session ID, timestamp and result. Full audit trail for every file change the AI made.

🌳

Porwr ffeil

Project tree panel for fast navigation. The AI uses list_project_files a list_open_files i ddarganfod llwybrau go iawn - nid yw byth yn dyfeisio lleoliadau ffeil.

Gorsaf Reoli'r Cwmni (CCS)

Rhedeg y Chameleon IDE fel canolbwynt swyddfa ar gyfer eich fflyd Logisteg 4 × 4 - nid oes angen meddalwedd anfon ar wahân

🗺

Gwthio llwybrau a POIs i yrwyr

Anfon a .gpx ffeil o'r DRhA yn uniongyrchol i unrhyw ffôn gyrrwr dros y sianel cyfoedion. Mae llwybrau'n llwytho ar unwaith ar fap byw y gyrrwr. Gellir gwthio setiau POI annibynnol - arosfannau tanwydd, peryglon, cyfeiriadau danfon - yn annibynnol ar unrhyw lwybr ac ymddangos fel pinnau â chôd lliw.

📦

Maniffest ac anfon anfoneb

Adeiladwch faniffest cargo neu paratowch anfoneb yn y DRhA, yna gwthiwch ef rhwng cyfoedion i ffôn y gyrrwr. Mae'r gyrrwr yn gweld y maniffest wedi'i lwytho ymlaen llaw - dim sganio, dim mynediad â llaw. Gellir llofnodi anfonebau ar y sgrin a'u dychwelyd dros yr un sianel.

🚨

Darllediad tocynwr fflyd

Anfonwch rybudd blaenoriaeth - Argyfwng, Pwysig, Cyfarwyddiadau neu Wybodaeth - ac mae'n ymddangos fel bar ticiwr lled llawn ar sgrin pob gyrrwr ar yr un pryd. Mae'r swyddfa yn cadw rheolaeth lawn ar sylw'r fflyd heb alwad ffôn i bob cerbyd.

📡

Gweinydd signal adeiledig - dim cyfnewid cwmwl

Gweinydd signal Chameleon (chameleon_signal.py) runs directly on the office PC alongside the IDE, or inside Termux on any Android phone in the fleet. Fleet phones connect to it over Tailscale or the local hotspot — no third-party relay, no monthly server bill.
