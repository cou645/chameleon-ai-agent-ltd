The AI Command Bridge

AI doesn't just chat — it operates the editor through a structured JSON protocol

📖

Read once, edit freely

The AI reads a file once with get_file_content, holds the content in context, then issues multiple anchor-based edits without re-reading between each change. No wasted round-trips.

🎯

Anchor-based replace_text

Edits reference exact text to find, not line numbers. Line numbers shift as code is added or removed — anchors don't. Multiple replace_text calls can be batched in a single response, all applied atomically.

📝

write_file for full rewrites

For new modules or complete rewrites the AI sends the entire file content in one command. The system automatically creates a .bak backup before overwriting and opens the result in a new editor tab.

💻

Terminal integration

The AI can run shell commands and receive the JSON result before continuing. It waits for each result, handles exit codes, and distinguishes diagnostic, configuration, destructive and session-ending commands automatically.

🛡️

Injection protection

All content returned from files, terminals and web pages is wrapped in [UNTRUSTED_CONTENT] markers. The AI is instructed to treat everything inside as data — never instructions — preventing prompt-injection attacks from malicious file content.

Editor Features

🗂️

Multi-tab code editor

Open multiple files simultaneously. Each tab tracks its own undo history, syntax highlighting and AI command context.

🔀

Built-in diff view

One click to compare the current editor state against the last saved version. Colour-coded unified diff with add/remove highlighting. Exit diff mode to return to full editing.

💬

Streaming AI chat

Responses stream token-by-token into the chat panel. IDE commands are extracted and executed in a background thread so the UI stays responsive throughout.

🗃️

Modular prompt system

System prompts are plain-text module files in ~/.chameleon/prompts/. Stack any combination: Tech Repair + Academic + Chat. Live token count and cost estimate per provider.

📜

Command history DB

Every AI command is logged to history.db with session ID, timestamp and result. Full audit trail for every file change the AI made.

🌳

File browser

Project tree panel for fast navigation. The AI uses list_project_files and list_open_files to discover real paths — it never invents file locations.

Voice & Speech

🎙️

Whisper speech-to-text

Local Whisper (whisper.cpp) transcribes your voice into the chat input. Hands-free coding — dictate questions, commands or whole prompts without touching the keyboard.

Company Command Station (CCS)

Run the Chameleon IDE as the office hub for your Logistics 4×4 fleet — no separate dispatch software needed

🗺

Push routes & POIs to drivers

Send a .gpx file from the IDE directly to any driver's phone over the peer channel. Routes load instantly on the driver's live map. Standalone POI sets — fuel stops, hazards, delivery addresses — can be pushed independently of any route and appear as colour-coded pins.

📦

Manifest & invoice dispatch

Build a cargo manifest or prepare an invoice in the IDE, then push it peer-to-peer to the driver's phone. The driver sees the manifest pre-loaded — no scanning, no manual entry. Invoices can be signed on-screen and returned over the same channel.

🚨

Fleet-wide ticker broadcast

Send a priority alert — Emergency, Important, Instructions or Info — and it appears as a full-width ticker bar on every driver's screen simultaneously. The office stays in full command of the fleet's attention without a phone call to each vehicle.

📡

Built-in signal server — no cloud relay

The Chameleon signal server (chameleon_signal.py) runs directly on the office PC alongside the IDE, or inside Termux on any Android phone in the fleet. Fleet phones connect to it over Tailscale or the local hotspot — no third-party relay, no monthly server bill.
