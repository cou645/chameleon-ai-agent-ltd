AI-Guided System Repair

The AI follows a structured surgical protocol — diagnose before touching, verify at every step

# Chameleon AI diagnosis sequence $ lsusb -v | testa -80 Bus 002 Dispositivo 003: ID 0bda:8153 Realtek USB 10/100/1000 LAN $ dmesg | tail -30 [12.445] r8152 2-1:1.0 eth1: rinominato da eth0 $ lsmod | grepr8152 r8152942080 $modinfo r8152 | versione grep versione: v1.12.13

Sequenza diagnostica strutturata

Hardware → kernel state → driver status → network → storage. The AI runs each step, reads the JSON result, then decides the next action. It never skips steps or makes assumptions.

🏥

Surgical repair protocol

Innanzitutto non distruttivo (ricarica il modulo, riavvia il servizio). Persistenza prima del rischio (configurazione istantanea). Verificare ad ogni passaggio. Ogni azione che richiede il riavvio attiva un Dead Man's Switch che imposta a @reboot riprendere lo script automaticamente.

🌐

Remote repair over Tailscale

Diagnose and repair machines anywhere on your Tailscale mesh. The AI issues commands on the remote machine via the terminal bridge, reads results and continues — even across reboots using the checkpoint/resume system.

Porta driver USB

Profile any USB device → analyse its protocol → generate a ready-to-install Linux driver

Sentiero A

Protocolli standard

Nessuna pagina del venditore trovata. Genera regole udev + configurazione modprobe. Nessuna compilazione richiesta. Funziona con HID, CDC, archiviazione di massa e altre classi standard.

udevmodprobe
Sentiero B

Protocollo fornitore parziale

Pagine del fornitore rilevate (≤2). Genera un demone dello spazio utente hidraw/uhid o un plugin ALSA. Funziona nello spazio utente: non è necessaria la compilazione del kernel.

Backend di acquisizione multipiattaforma

ETW on Windows, IOKit on macOS, usbmon on Linux. Profile on any OS and generate a Linux driver from the resulting profile.json.

🍓

Compilazione incrociata di Raspberry Pi

Obiettivi del makefile cross-aarch64 (Tav. 4/5) e cross-armv7 (Pi 3) sono inclusi automaticamente nel percorso C. Compila su x86 e distribuisci su ARM.

📦

Confezione DKMS

Path C drivers are packaged for DKMS so they survive kernel upgrades without manual recompilation. The AI reviews analysis.json and prompts for any remaining TODO stubs.
