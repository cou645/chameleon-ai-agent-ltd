The AI Command Bridge

AI doesn't just chat — it operates the editor through a structured JSON protocol

📖

Citiți o dată, editați liber

The AI reads a file once with get_file_content, menține conținutul în context, apoi emite mai multe editări bazate pe ancoră fără a reciti între fiecare modificare. Fără călătorii dus-întors pierdute.

🎯

replace_text bazat pe ancora

Editările fac referire la textul exact de găsit, nu numerele de rând. Numerele liniilor se schimbă pe măsură ce codul este adăugat sau eliminat - ancorele nu. Multiplu replace_text apelurile pot fi grupate într-un singur răspuns, toate aplicate atomic.

📝

write_file pentru rescrieri complete

For new modules or complete rewrites the AI sends the entire file content in one command. The system automatically creates a .bak backup înainte de suprascriere și deschide rezultatul într-o nouă filă editor.

💻

Integrarea terminalelor

The AI can run shell commands and receive the JSON result before continuing. It waits for each result, handles exit codes, and distinguishes diagnostic, configuration, destructive and session-ending commands automatically.

🛡️

Protectie la injectie

Tot conținutul returnat de la fișiere, terminale și pagini web este împachetat [UNTRUSTED_CONTENT] markers. The AI is instructed to treat everything inside as data — never instructions — preventing prompt-injection attacks from malicious file content.

Caracteristici editor

🗂️

Editor de cod cu mai multe file

Open multiple files simultaneously. Each tab tracks its own undo history, syntax highlighting and AI command context.

🔀

Vizualizare diferită încorporată

One click to compare the current editor state against the last saved version. Colour-coded unified diff with add/remove highlighting. Exit diff mode to return to full editing.

💬

Streaming AI chat

Răspunsurile transmit token-cu-token în panoul de chat. Comenzile IDE sunt extrase și executate într-un thread de fundal, astfel încât interfața de utilizare rămâne receptivă pe tot parcursul.

🗃️

Sistem modular prompt

System prompts are plain-text module files in ~/.chameleon/prompts/. Stack any combination: Tech Repair + Academic + Chat. Live token count and cost estimate per provider.

📜

Istoricul comenzilor DB

Every AI command is logged to history.db with session ID, timestamp and result. Full audit trail for every file change the AI made.

🌳

Browser de fișiere

Project tree panel for fast navigation. The AI uses list_project_files şi list_open_files pentru a descoperi căi reale — nu inventează niciodată locațiile fișierelor.

Voce și vorbire

🎙️

Whisper speech-to-text

Local Whisper (whisper.cpp) transcribes your voice into the chat input. Hands-free coding — dictate questions, commands or whole prompts without touching the keyboard.

Aplicație mobilă — releu de chat și control prezentator

Stația de comandă a companiei (CCS)

Rulați Chameleon IDE ca hub de birou pentru flota dvs. Logistics 4×4 - nu este nevoie de software de expediere separat

🗺

Împingeți rutele și punctele de interes către șoferi

Trimite un .gpx fișier din IDE direct pe telefonul oricărui șofer prin canalul peer. Rutele se încarcă instantaneu pe harta live a șoferului. Seturile de puncte de interes autonome — opriri de combustibil, pericole, adrese de livrare — pot fi împinse independent de orice rută și pot apărea ca pini cu coduri de culoare.

📦

Expediere manifest și factură

Creați un manifest de marfă sau pregătiți o factură în IDE, apoi împingeți-o peer-to-peer la telefonul șoferului. Driverul vede manifestul preîncărcat — fără scanare, fără introducere manuală. Facturile pot fi semnate pe ecran și returnate pe același canal.

🚨

Ticker difuzat la nivelul întregii flote

Trimiteți o alertă prioritară — Urgență, Important, Instrucțiuni sau Informații — și apare simultan ca o bară de ticker pe toată lățimea pe ecranul fiecărui șofer. Biroul rămâne în deplină control asupra atenției flotei fără un apel telefonic la fiecare vehicul.

📡

Server de semnal încorporat — fără releu cloud

Serverul de semnal Cameleon (chameleon_signal.py) runs directly on the office PC alongside the IDE, or inside Termux on any Android phone in the fleet. Fleet phones connect to it over Tailscale or the local hotspot — no third-party relay, no monthly server bill.
