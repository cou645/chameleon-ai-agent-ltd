KameleonStack

The universal content container. Each stack holds multiple slots (images, text, PDFs). Two modes:

  • コンテンツ — 教師が管理するスライド
  • ステージ — ピアプッシュの送信を受け入れます
🎬

アニメーションシステム

フリーハンド モーション パスを使用したオブジェクトごとの出入りアニメーション。パスは画面上に描画され、保存前に Douglas-Peucker 圧縮と Catmull-Rom スプライン補間によって平滑化されます。

🗃️

ドレッサーパネル

オフステージ待機エリア — ドレッサーでオブジェクトを作成し、ダブルクリックしてステージ上でプレビューします。右クリック メニューから [ドレッサー] にドラッグして戻します。 [プロパティ] パネルには、アニメーションの開始/終了状態が表示されます。

🔢

番号付きキュー

トリガー ポイントは、劇場の照明の合図と同じように、プレゼンテーションを段階的に進めます。各キューは、オブジェクトの内外をアニメートしたり、スタック スロットを切り替えたり、時間指定されたシーケンスを実行したりできます。

💾

.kpt プロジェクト バンドル

プロジェクトは ZIP アーカイブとして保存されます (.kpt) containing all media assets, layout data and animation configs. Open on any machine running Chameleon.

📐

論理座標

ステージは、ディスプレイ解像度に関係なく、0 ～ 10,000 の論理座標空間を使用します。プレゼンテーションはラップトップの画面でも 4K プロジェクターでも同じように見えます。

モバイルとピアコントロール

携帯電話からプレゼンターを実行します。生徒が自分のコンテンツからコンテンツをプッシュできるようにする

📱

Android mobile bridge

The Chameleon mobile bridge (WebSocket) relays theatre commands from any Android device. Control stack_next / prev / 再生 / 一時停止 / 停止、画像またはテキスト スロットをスタックにプッシュし、リアルタイムでスタック状態のブロードキャストを確認します。

🙋

ピアの提出物

Students push images or text to a Stage-mode stack from their phones. Submissions appear in the thumbnail strip with an orange pulsing border. The presenter can 承認、保留、拒否、または昇格 画面上でライブになる前に、それぞれを確認します。

✏️

共有製図板

モバイルブリッジから制御可能な共同フリーハンドキャンバス — 描画ストローク / クリア / 元に戻す. Ideal for annotating slides, brainstorming or live quizzes.

🤖

ADB 電話ミラーと PPT キャプチャ

ADBフォンボックス streams a live Android screen directly onto the stage via ADB. PPTミラーボックス captures a PowerPoint window — present your existing decks inside Kameleon without conversion.
