IDE alimenté par l'IA

Un éditeur de code PyQt6 complet dans lequel l'IA ne se contente pas de suggérer : il lit, écrit, exécute et vérifie le code en votre nom via un pont de commandes structuré.

IA multi-fournisseurs 🎙️ Saisie vocale et TTS 🔀 Vue différentielle ⚡ Réponses en streaming

Le pont de commandement de l'IA

L'IA ne se contente pas de discuter : elle fait fonctionner l'éditeur via un protocole JSON structuré

📖

Lisez une fois, modifiez librement

L'IA lit un fichier une fois avec get_file_content, maintient le contenu dans son contexte, puis émet plusieurs modifications basées sur des ancres sans relire entre chaque modification. Pas d’allers-retours inutiles.

🎯

replace_text basé sur une ancre

Modifie le texte exact de référence à rechercher, pas les numéros de ligne. Les numéros de ligne changent à mesure que du code est ajouté ou supprimé, ce qui n'est pas le cas des ancres. Multiple replace_text les appels peuvent être regroupés en une seule réponse, tous appliqués de manière atomique.

📝

write_file pour les réécritures complètes

Pour les nouveaux modules ou les réécritures complètes, l'IA envoie l'intégralité du contenu du fichier en une seule commande. Le système crée automatiquement un .bak sauvegarde avant d'écraser et ouvre le résultat dans un nouvel onglet d'éditeur.

💻

Intégration des terminaux

L'IA peut exécuter des commandes shell et recevoir le résultat JSON avant de continuer. Il attend chaque résultat, gère les codes de sortie et distingue automatiquement les commandes de diagnostic, de configuration, de destruction et de fin de session.

🛡️

Protection contre les injections

Tout le contenu renvoyé par les fichiers, les terminaux et les pages Web est enveloppé dans [UNTRUSTED_CONTENT] marqueurs. L’IA est chargée de traiter tout ce qui se trouve à l’intérieur comme des données – jamais d’instructions – pour empêcher les attaques par injection rapide provenant de contenus de fichiers malveillants.

Fonctionnalités de l'éditeur

🗂️

Éditeur de code multi-onglets

Ouvrez plusieurs fichiers simultanément. Chaque onglet suit son propre historique d'annulation, sa coloration syntaxique et son contexte de commande AI.

🔀

Vue différentielle intégrée

Un clic pour comparer l'état actuel de l'éditeur avec la dernière version enregistrée. Diff unifié à code couleur avec ajout/suppression de surbrillance. Quittez le mode diff pour revenir à l’édition complète.

💬

Chat IA en streaming

Les réponses sont diffusées jeton par jeton dans le panneau de discussion. Les commandes IDE sont extraites et exécutées dans un thread en arrière-plan afin que l'interface utilisateur reste réactive tout au long.

🗃️

Système d'invite modulaire

Les invites système sont des fichiers de module en texte brut dans ~/.chameleon/prompts/. Empilez n'importe quelle combinaison : Réparation technique + Académique + Chat. Nombre de jetons en direct et estimation des coûts par fournisseur.

📜

Base de données d'historique des commandes

Chaque commande AI est enregistrée dans history.db avec ID de session, horodatage et résultat. Piste d'audit complète pour chaque modification de fichier effectuée par l'IA.

🌳

Navigateur de fichiers

Panneau d'arborescence du projet pour une navigation rapide. L'IA utilise list_project_files et list_open_files pour découvrir de vrais chemins - il n'invente jamais les emplacements de fichiers.

Voix et parole

🎙️

Synthèse parole-texte chuchotée

Local Whisper (whisper.cpp) transcrit votre voix dans la saisie du chat. Codage mains libres : dictez des questions, des commandes ou des invites entières sans toucher au clavier.

Synthèse vocale Piper

Les réponses de l'IA sont lues à haute voix à l'aide de Piper TTS : des voix rapides, locales et naturelles. Idéal pour l'accessibilité ou pour travailler les mains occupées. Les chemins de plateforme sont résolus automatiquement sous Linux, Windows et macOS.

Station de commandement de la compagnie (CCS)

Exécutez l'IDE Chameleon en tant que plateforme de bureau pour votre flotte logistique 4×4 – aucun logiciel de répartition séparé n'est nécessaire

🗺

Envoyer des itinéraires et des POI aux conducteurs

Envoyez un .gpx fichier de l'IDE directement sur le téléphone de n'importe quel conducteur via le canal homologue. Les itinéraires se chargent instantanément sur la carte en direct du conducteur. Des ensembles de POI autonomes (arrêts de carburant, dangers, adresses de livraison) peuvent être poussés indépendamment de tout itinéraire et apparaissent sous forme d'épingles à code couleur.

📦

Envoi des manifestes et factures

Créez un manifeste de fret ou préparez une facture dans l'IDE, puis envoyez-le en peer-to-peer sur le téléphone du conducteur. Le conducteur voit le manifeste préchargé – pas de numérisation, pas de saisie manuelle. Les factures peuvent être signées à l’écran et renvoyées via le même canal.

🚨

Diffusion du téléscripteur à l'échelle de la flotte

Envoyez une alerte prioritaire (Urgence, Important, Instructions ou Informations) et elle apparaît simultanément sous la forme d'une barre de défilement pleine largeur sur l'écran de chaque conducteur. Le bureau reste à la pointe de l'attention de la flotte sans appel téléphonique à chaque véhicule.

📡

Serveur de signaux intégré – pas de relais cloud

Le serveur de signaux Caméléon (chameleon_signal.py) s'exécute directement sur le PC du bureau aux côtés de l'IDE, ou dans Termux sur n'importe quel téléphone Android de la flotte. Les téléphones de flotte s'y connectent via Tailscale ou le point d'accès local – pas de relais tiers, pas de facture mensuelle de serveur.
