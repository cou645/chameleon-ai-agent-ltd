The AI Command Bridge

AI doesn't just chat — it operates the editor through a structured JSON protocol

📖

Przeczytaj raz, edytuj swobodnie

The AI reads a file once with get_file_content, utrzymuje treść w kontekście, a następnie wprowadza wiele zmian opartych na zakotwiczeniach bez ponownego czytania pomiędzy każdą zmianą. Żadnych zbędnych podróży w obie strony.

🎯

Tekst zamiany oparty na kotwicy

Zmiany odwołują się do szukanego tekstu, a nie do numerów wierszy. Numery linii zmieniają się wraz z dodawaniem lub usuwaniem kodu – kotwice nie. Wiele replace_text wywołania mogą być grupowane w jedną odpowiedź, wszystkie stosowane atomowo.

📝

write_file dla pełnego przepisywania

For new modules or complete rewrites the AI sends the entire file content in one command. The system automatically creates a .bak kopię zapasową przed nadpisaniem i otwiera wynik w nowej karcie edytora.

💻

Integracja terminala

The AI can run shell commands and receive the JSON result before continuing. It waits for each result, handles exit codes, and distinguishes diagnostic, configuration, destructive and session-ending commands automatically.

🛡️

Zabezpieczenie wtrysku

Cała zawartość zwracana z plików, terminali i stron internetowych jest opakowana [UNTRUSTED_CONTENT] markers. The AI is instructed to treat everything inside as data — never instructions — preventing prompt-injection attacks from malicious file content.

Funkcje edytora

🗂️

Edytor kodu z wieloma zakładkami

Open multiple files simultaneously. Each tab tracks its own undo history, syntax highlighting and AI command context.

🔀

Wbudowany widok różnicowy

One click to compare the current editor state against the last saved version. Colour-coded unified diff with add/remove highlighting. Exit diff mode to return to full editing.

💬

Streaming AI chat

Odpowiedzi przesyłane są token po tokenie do panelu czatu. Polecenia IDE są wyodrębniane i wykonywane w wątku w tle, dzięki czemu interfejs użytkownika pozostaje responsywny przez cały czas.

🗃️

Modułowy system podpowiedzi

System prompts are plain-text module files in ~/.chameleon/prompts/. Stack any combination: Tech Repair + Academic + Chat. Live token count and cost estimate per provider.

📜

Historia poleceń DB

Every AI command is logged to history.db with session ID, timestamp and result. Full audit trail for every file change the AI made.

🌳

Przeglądarka plików

Project tree panel for fast navigation. The AI uses list_project_files I list_open_files do odkrywania prawdziwych ścieżek — nigdy nie wymyśla lokalizacji plików.

Głos i mowa

🎙️

Whisper speech-to-text

Local Whisper (whisper.cpp) transcribes your voice into the chat input. Hands-free coding — dictate questions, commands or whole prompts without touching the keyboard.

Stacja dowodzenia kompanii (CCS)

Uruchom Chameleon IDE jako centrum biurowe dla swojej floty Logistics 4×4 — nie jest potrzebne osobne oprogramowanie wysyłkowe

🗺

Przesyłaj kierowcom trasy i punkty POI

Wyślij .gpx plik z IDE bezpośrednio do telefonu dowolnego kierowcy za pośrednictwem kanału równorzędnego. Trasy ładują się natychmiast na aktualnej mapie kierowcy. Samodzielne zestawy POI — przystanki paliwowe, zagrożenia, adresy dostaw — można przesuwać niezależnie od dowolnej trasy i wyświetlać jako oznaczone kolorami pinezki.

📦

Manifest i wysyłka faktury

Zbuduj manifest ładunku lub przygotuj fakturę w IDE, a następnie wypchnij ją peer-to-peer na telefon kierowcy. Kierowca widzi wstępnie załadowany manifest — bez skanowania i ręcznego wprowadzania. Faktury można podpisywać na ekranie i zwracać tym samym kanałem.

🚨

Transmisja giełdowa obejmująca całą flotę

Wyślij alert priorytetowy — Alarm, Ważne, Instrukcje lub Informacje — a pojawi się on jednocześnie w postaci paska o pełnej szerokości na ekranie każdego kierowcy. Biuro w pełni kontroluje całą flotę, bez konieczności wykonywania połączeń telefonicznych do każdego pojazdu.

📡

Wbudowany serwer sygnału — brak przekaźnika w chmurze

Serwer sygnałowy Chameleon (chameleon_signal.py) runs directly on the office PC alongside the IDE, or inside Termux on any Android phone in the fleet. Fleet phones connect to it over Tailscale or the local hotspot — no third-party relay, no monthly server bill.
