AI-Powered IDE

A full-featured PyQt6 code editor where AI doesn't just suggest — it reads, writes, runs and verifies code on your behalf through a structured command bridge.

The AI Command Bridge

AI doesn't just chat — it operates the editor through a structured JSON protocol

📖

اقرأ مرة واحدة، وحرر بحرية

The AI reads a file once with get_file_content، يحتفظ بالمحتوى في سياقه، ثم يصدر تعديلات متعددة مستندة إلى نقطة الارتساء دون إعادة القراءة بين كل تغيير. لا تضيع رحلات الذهاب والإياب.

🎯

استبدال_نص قائم على المرساة

تشير التعديلات إلى النص الدقيق المطلوب العثور عليه، وليس أرقام الأسطر. تتغير أرقام الأسطر عند إضافة التعليمات البرمجية أو إزالتها - ولا تتغير نقاط الارتساء. عديد replace_text يمكن تجميع المكالمات في رد واحد، ويتم تطبيقها جميعًا بشكل تلقائي.

📝

write_file لإعادة الكتابة الكاملة

For new modules or complete rewrites the AI sends the entire file content in one command. The system automatically creates a .bak النسخ الاحتياطي قبل الكتابة فوق ويفتح النتيجة في علامة تبويب محرر جديدة.

💻

التكامل الطرفي

The AI can run shell commands and receive the JSON result before continuing. It waits for each result, handles exit codes, and distinguishes diagnostic, configuration, destructive and session-ending commands automatically.

🛡️

حماية الحقن

يتم تغليف جميع المحتويات التي يتم إرجاعها من الملفات والمحطات الطرفية وصفحات الويب [UNTRUSTED_CONTENT] markers. The AI is instructed to treat everything inside as data — never instructions — preventing prompt-injection attacks from malicious file content.

ميزات المحرر

🗂️

محرر كود متعدد علامات التبويب

Open multiple files simultaneously. Each tab tracks its own undo history, syntax highlighting and AI command context.

🔀

المدمج في عرض الفرق

One click to compare the current editor state against the last saved version. Colour-coded unified diff with add/remove highlighting. Exit diff mode to return to full editing.

💬

Streaming AI chat

يتم بث الردود رمزًا تلو الآخر في لوحة الدردشة. يتم استخراج أوامر IDE وتنفيذها في سلسلة محادثات في الخلفية بحيث تظل واجهة المستخدم مستجيبة طوال الوقت.

🗃️

نظام موجه وحدات

System prompts are plain-text module files in ~/.chameleon/prompts/. Stack any combination: Tech Repair + Academic + Chat. Live token count and cost estimate per provider.

📜

سجل الأوامر DB

Every AI command is logged to history.db with session ID, timestamp and result. Full audit trail for every file change the AI made.

🌳

متصفح الملفات

Project tree panel for fast navigation. The AI uses list_project_files و list_open_files لاكتشاف مسارات حقيقية - فهو لا يخترع أبدًا مواقع الملفات.

الصوت والكلام

🎙️

Whisper speech-to-text

Local Whisper (whisper.cpp) transcribes your voice into the chat input. Hands-free coding — dictate questions, commands or whole prompts without touching the keyboard.

محطة قيادة الشركة (CCS)

قم بتشغيل Chameleon IDE كمركز مكتبي لأسطولك اللوجستي 4×4 - لا حاجة إلى برنامج إرسال منفصل

🗺

طرق الدفع والنقاط المهمة للسائقين

أرسل أ .gpx الملف من IDE مباشرة إلى هاتف أي سائق عبر قناة النظير. يتم تحميل المسارات على الفور على الخريطة المباشرة للسائق. يمكن دفع مجموعات النقاط المهمة المستقلة - محطات الوقود، والمخاطر، وعناوين التسليم - بشكل مستقل عن أي طريق وتظهر كدبابيس مرمزة بالألوان.

📦

البيان وإرسال الفاتورة

أنشئ بيان شحنة أو قم بإعداد فاتورة في IDE، ثم أرسلها من نظير إلى نظير إلى هاتف السائق. يرى السائق أن البيان تم تحميله مسبقًا - لا يوجد مسح ولا إدخال يدوي. يمكن توقيع الفواتير على الشاشة وإعادتها عبر نفس القناة.

🚨

بث شريط على مستوى الأسطول

أرسل تنبيهًا ذا أولوية - حالة طوارئ أو هام أو تعليمات أو معلومات - وسيظهر كشريط شريط كامل العرض على شاشة كل سائق في وقت واحد. ويظل المكتب مسيطرًا بشكل كامل على انتباه الأسطول دون إجراء مكالمة هاتفية مع كل مركبة.

📡

خادم إشارة مدمج — لا يوجد مرحل سحابي

خادم إشارة الحرباء (chameleon_signal.py) runs directly on the office PC alongside the IDE, or inside Termux on any Android phone in the fleet. Fleet phones connect to it over Tailscale or the local hotspot — no third-party relay, no monthly server bill.
